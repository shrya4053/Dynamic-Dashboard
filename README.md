# Dynamic-Dashboard
Prepared a dynamic dashboard that tells sales by region of electronic products.

## Project Objective
To practice a dynamic dashboard that changes with the inout.

## Dataset Used
<a href= "https://github.com/shrya4053/Dynamic-Dashboard/blob/main/dynamic%20dashboard.xlsx"> Dataset</a>

## Questions
- Which region generates the highest percentage of bike purchases?
- Which seller sells which item the most in which region?
- Which items are purchased the most in which region?

## Dashboard
<a href= https://github.com/shrya4053/Dynamic-Dashboard/blob/main/Screenshot%202025-11-28%20190842.png> View Dashboard</a>

## Process
  - Verified data for any missing values and anomalities, and sort out the same.
  - Made sure data is consisted and clean with respect to data type, data format and values used.
  - Created pivot tables according to the questions asked.
  - Merged all pivot tables into one dashboard and applied slicer to make it dynamic.
  - Added pie charts to visualise the insights better.

## Project Insight
- Comapy sells most of the items like CPU, Hard disk in north region the most and a few product in east 
- Top 3 selling items are mouse, keyboard and printer in all the regions. 
- Top 3 sellers are PC ZOne, Abc Enterprise and Super PC, whereas Ap Enterprise brought least sale.

## Conclusion
- The company should focus on good performing regions to sell more items. Also the company should plan to promote thier items in the East and West regions as well, either by ads or investing more to make the products according to the needs of the people of thst region.
- They should focus on top selling products and also extra care on rest other products.
- They should invest more on top sellers and give them extra perks and benefits wehreas give the least performing one a particular target on monthly or yearly basis to achieve the perks.
